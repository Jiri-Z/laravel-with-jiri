### Phase 1 — HIGH Priority: Quiz Logic Duplication & Safety

- [ ] 1.1 Extract shared `AnswerChecker` service
  - Move `checkSingle()`, `checkMultiple()`, `checkText()` logic from both `SubmitQuizAnswer` and `TriviaQuiz` into `App\Services\AnswerChecker`
  - Both callers use the same service (TDD: write AnswerCheckerTest first, then refactor callers)
  - Unify normalization: use `mb_strtolower(trim(...))` consistently for text answers

- [ ] 1.2 Fix race condition handler in `SubmitQuizAnswer`
  - Catch only duplicate-key `QueryException` (check `$e->getCode()` for PostgreSQL `'23505'` or SQLite `19`)
  - Re-throw all other `QueryException` instances
  - Test: duplicate submission returns existing result; non-duplicate DB error propagates

- [ ] 1.3 Consistent action invocation pattern
  - Choose one convention: either all invokable (`__invoke`) or all `handle()`
  - Convert `SwitchLocale` to a handle-based action class for consistency
  - Or convert `MarkStepComplete` and `EnrollInCourse` to invokable — pick one pattern

- [ ] 1.4 Fix `QuizViewer` json_decode without `true` flag
  - Add `true` second argument to `json_decode()` in `QuizViewer::mount()` line 71
  - Test: multiple-answer re-mount restores answers as arrays, not stdClass
